FETCH RULES
1. Single batched HTTP request per refresh cycle (no per-token calls).
2. Endpoint (FreeCryptoAPI preferred):
  GET https://api.freecryptoapi.com/v1/getDataCurrency?symbol=WBTC+ETH+USDC+DAI+UNI+...&local=USD
   (All symbols space or plus-separated in one symbol param; use '+' in actual URL encoding.)
3. Cache server-side 30–60s. Reject client attempts to pass custom symbols.
4. Return partial data even if some symbols fail. Never throw for a single-symbol failure.

ERROR HANDLING
1. For any symbol missing in provider response: priceUSD = null, add field sourceError with provider status/message.
2. Preserve successfully returned symbols.
3. Log raw provider payload + ISO timestamp for observability & quota tracking.

SECURITY & VALIDATION
1. This list is CLOSED. Ignore any user-supplied extra symbols.
2. Validate address format (EIP-55 checksum or lower-case acceptable) before on-chain actions.
3. Consider on first startup: verify each address against authoritative explorer (Etherscan) and record a hash of the list for tamper detection.
